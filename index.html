one command to install. one local cockpit to run.

HTTP sidecar for chrome auth prompts, SOCKS5 for everything else.

installer downloads a prebuilt binary from GitHub releases. your domain hosts the installer and landing page.

what it does
  • SSHs into your VPS, installs what's needed, configures everything
  • HTTP proxy (squid + auth) or SOCKS5 (microsocks)
  • HTTP sidecar mode that won't overwrite an existing squid config
  • auto-generates credentials, rotates them when you want
  • firewall rules handled (or skipped, your call)
  • saved ships (local profiles) and hangars (remote setup)
  • destroy hangar to clean up after yourself like a responsible adult
quick start

1. you need a VPS (debian/ubuntu, root ssh)

2. install beammeup with the one-liner above

3. run beammeup, create a ship, and launch. that's it.

actions
show display current setup and (when available) credentials
configure install or repair (keeps existing creds)
rotate new credentials, who dis
destroy destroy hangar (remove beammeup remote config)
ships + hangars

ship = local profile in ~/.beammeup/ships/. it stores host/user/ports and defaults.

hangar = the remote services and metadata on that VPS (/etc/beammeup/).

ship files never store ssh passwords. beammeup prompts at runtime and caches in-memory per session.

cli usage
beammeup interactive cockpit
beammeup --list-ships see what you've got
beammeup --ship istanbul open a saved ship
beammeup --self-update beam up a fresh release

target: debian/ubuntu vps with root ssh and systemd.

no telemetry. no accounts. no nonsense. just a local tool doing honest work.
